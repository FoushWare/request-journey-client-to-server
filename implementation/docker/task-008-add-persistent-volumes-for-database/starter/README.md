# Starter: Add Persistent Volumes for Database

This directory contains the starting point for adding persistent volumes.

## Current State

- Database running in container
- Data lost when container is removed
- No data persistence

## Your Task

1. Create named volume for database
2. Mount volume to database container
3. Test data persistence (restart container, verify data)
4. Backup and restore volume
